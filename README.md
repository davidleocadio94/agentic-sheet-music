# agentic-sheet-music

PDF in → annotated PDF + per-measure playback server out.

OMR is **Gemini Vision only** (as of 2026-04-25 — the previous Audiveris hybrid was burned down). Quality is governed by an eval-driven autoresearch loop that exits only when the score reaches 100%.

## Pipeline

```
PDF
 │
 ▼
Gemini 3.1 Pro Vision  ──→  MusicXML  ──→  harmony analysis
                                        │
                                        ├──→  annotated PDF (re-engraved via Verovio)
                                        │
                                        └──→  playalong server (per-measure click, tempo slider)
```

## Eval loop

```
eval-fixtures/
├── 01-pitch/    02-rhythm/    03-meter/    04-key/
└── 05-voicing/  06-articulation/  07-real-pieces/
```

Each fixture: `ground-truth.musicxml` (hand-written) + `source.pdf` (auto-generated by Verovio from GT).

The metric is per-measure exact-match: pitches + rhythms + voices identical to GT. **Target: 100%.**

```
uv run eval                    # one shot
uv run eval --refresh-pdfs     # rebuild source PDFs from GT
uv run improve-omr             # the autoresearch loop (no iteration cap)
```

## Agents (.claude/agents/)

- `eval-curator` — adds new fixtures
- `omr-experimenter` — proposes/runs experiments to improve the score
- `failure-diagnoser` — clusters errors, recommends next experiment
- `ground-truth-builder` — helps user produce GT for real-piece fixtures

## Skills (.claude/skills/)

- `improve-omr` — orchestrates the autoresearch loop

## Constraints

- Repo size ≤ 1 GB (currently well under)
- Source PDFs in `eval-fixtures/` are gitignored; regenerate with `--refresh-pdfs`
- Never modify the evaluator mid-loop (would change the metric)
- Never modify fixtures via experiments (only via `eval-curator`)
